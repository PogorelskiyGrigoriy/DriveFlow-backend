# на странице логина тумблер "я - учитенль". логин (телефон?) + пароль

# 1. Главный экран: Календарь на день (Daily Schedule)
Это стартовый экран после логина.
Вместо сложного недельного расписания на мобильном телефоне лучше всего показать линейный список часов на сегодня.

Быстрые действия: По клику на занятый слот всплывает меню:
1. Выдать статус COMPLETED (Урок проведен).
2. Выдать статус CANCELLED (Урок отменен).
3. Позвонить ученику (прямой вызов tel:).

# 2. Экран «Мои Ученики» (Student Management)
Простой упорядоченный список всех учеников, закрепленных за этим инструктором.
1. Главная фича — Кнопка «Добавить ученика»:
Открывает модальное окно с тремя полями: Имя, Фамилия, Номер телефона. Учитель вводит данные, жмет «Сохранить», бэкенд создает пользователя со статусом STUDENT.
2. Триггер рассылки («Та самая кнопка»):
Напротив каждого ученика в списке должна быть кнопка со значком сообщения.
По клику на неё бэкенд генерирует MagicToken для этого ученика и «отправляет» SMS/WhatsApp со ссылкой на вход.
На этапе MVP система пишет в консоль бэкенда готовый лог, который ты сможешь показать другу на презентации.

# 3. Экран «Настройки графика» (Availability Settings)
Интерфейс: 6 строчек (с воскресенья по четверг).
Для каждого дня недели: Чекбокс «Рабочий день» + два выпадающих списка (Dropdown) для выбора часов:
Начало: (выбор от 06:00 до 12:00)
Конец: (выбор от 13:00 до 22:00)
По клику на кнопку «Сохранить» данные улетают в таблицу InstructorAvailability через @@unique апдейт.
